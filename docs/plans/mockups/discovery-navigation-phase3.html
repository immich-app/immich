After clicking a year, the month grid expands below showing 12 months with counts. A breadcrumb trail allows navigating back to the year level.

Design notes: When filters are active, the temporal picker reflects the filtered result set. Years with zero matching photos are greyed out. The volume bars rescale to the filtered maximum. This happens automatically because the GET /timeline/buckets endpoint already supports all filter parameters — the picker just passes the active filters through. The header count also updates to show "342 photos (filtered)" instead of the total.
4. Current Scrubber vs. Temporal Picker
The existing scrubber (left) maps the entire archive to a thin vertical bar. The temporal picker (right) provides discrete, countable navigation targets.
Current: Scrubber

Each tick = 1 month

20 years = 240 ticks

~2.5px per month on screen

Precise navigation requires sub-pixel accuracy on touch devices

Phase 3: Temporal Picker

2 clicks to any point in a 20-year archive

Click targets: 72px+ wide buttons

Counts visible before clicking

Volume bars show activity distribution

Works identically on desktop and mobile

Implementation note: The temporal picker does not replace the scrubber — it complements it. The scrubber remains available for fine-grained scrolling. The picker provides coarse-grained jumping. Users can use either or both. No server changes needed — year counts are aggregated client-side from existing GET /timeline/buckets monthly data.
