# Global Face Identities: Repair Performance Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for every code task. Use superpowers:verification-before-completion before claiming this phase is done. This is phase 5 of the global face identities roadmap and depends on phases 1-4.

**Goal:** Add safe same-person and detach repair flows, then harden the people/filter/search query shapes and run the full release verification for global face identities.

**Architecture:** Manual repair works through scoped profile references, not raw identity ids. The server resolves those references under the actor's access, rejects global identity merges that would mutate inaccessible attached profiles, and uses detach to split one scoped profile into a fresh identity. Performance hardening verifies every request-time people/filter/search path applies access before grouping, paginates, avoids vector matching, and uses identity indexes.

**Tech Stack:** NestJS, Kysely, `@immich/sql-tools`, Vitest small and medium tests, generated OpenAPI SDK, Svelte 5 detail-page tests, PostgreSQL `EXPLAIN`, generated SQL snapshots.

---

## Scope

This phase owns:

- Manual same-person repair endpoint and SDK.
- Manual detach endpoint and SDK.
- Access checks for repair references.
- Web repair actions on personal and space person detail pages.
- Query-shape and generated SQL review for `/people`, filter suggestions, identity token filtering, people search, and global search.
- Full backend, frontend, OpenAPI, and leak checks for all global face identity phases.

This phase does not own:

- A general admin identity browser.
- Cross-user repair of profiles the actor cannot access.
- A denormalized read model unless measured query plans prove the access-scoped query shape cannot meet performance targets.
- Merge-history based partial split recovery. Detach creates a fresh identity for the selected scoped profile; full historical split can be a future feature.

## TDD Rules

- Add repair permission tests before adding endpoints.
- Read access is not enough for repair writes. Personal refs require ownership; space-person refs require owner/editor role in that space.
- A global identity merge is allowed only when every scoped profile attached to each involved identity is accessible for repair by the actor. If not, reject the merge; the actor can detach an accessible scoped profile instead.
- Do not merge names, aliases, hidden state, favorite state, thumbnails, or birth dates across scoped profiles during repair. Identity graph changes only.
- Detach must move only the selected scoped profile and its faces to a fresh identity.
- Detach must not move `face_identity_face` rows that also back a non-repairable personal profile or space profile. Reject the detach until a future scoped-exclusion repair model exists.
- Request-time people/filter/search paths must not run vector similarity.
- If an `EXPLAIN` shows unbounded hydration or unacceptable scans on representative data, add indexes or split the query before release.

Add service methods that call repository methods and enforce high-level checks:

```ts
async mergeScopedPeople(auth: AuthDto, dto: MergeScopedPeopleDto): Promise<void> {
  const resolved = await this.faceIdentityRepository.resolveRepairRefs(auth.user.id, dto);
  if (!resolved.accessible) {
    throw new BadRequestException('One or more people were not found or are not accessible');
  }
  if (!resolved.allAttachedProfilesRepairable) {
    throw new ForbiddenException('Cannot merge identities with inaccessible attached profiles');
  }
  await this.faceIdentityRepository.mergeIdentities({
    targetIdentityId: resolved.targetIdentityId,
    sourceIdentityIds: resolved.sourceIdentityIds,
    source: 'manual',
  });
}

async detachScopedPerson(auth: AuthDto, dto: DetachScopedPersonDto): Promise<void> {
  const resolved = await this.faceIdentityRepository.resolveDetachRef(auth.user.id, dto.profile);
  if (!resolved.accessible) {
    throw new BadRequestException('Person was not found or is not accessible');
  }
  if (!resolved.allBackingFacesRepairable) {
    throw new ForbiddenException('Cannot detach a profile whose faces also back inaccessible profiles');
  }
  await this.faceIdentityRepository.detachScopedProfile(dto.profile);
}
```

Access rules:

- `person` ref: actor must own the `person` row.
- `space-person` ref: actor must be owner or editor of `spaceId`.
- refs must resolve to non-null `identityId`.
- all resolved identities must have compatible type/species rules.
- `allAttachedProfilesRepairable` is true only when every `person` and `shared_space_person` attached to each involved identity passes the same repair access rules for the actor.
- `allBackingFacesRepairable` is true only when every `asset_face` that would move during detach belongs to a personal or space profile the actor can repair.

- [ ] **Step 3: Implement detach**

Add:

```ts
detachScopedProfile(profile: ScopedPersonProfileRefDto): Promise<string>;
```

Detach algorithm:

1. Create a fresh `face_identity`.
2. Update the selected `person.identityId` or `shared_space_person.identityId` to the fresh identity.
3. Move `face_identity_face` rows for faces attached to the selected scoped profile to the fresh identity.
4. For a `person` profile, move faces where `asset_face.personId = profile.id`.
5. For a `space-person` profile, move faces joined through `shared_space_person_face.personId = profile.id`.
6. Before moving a face, verify that moving it will not mutate a non-repairable backing personal profile or space profile; reject the detach if any backing face is unsafe.
7. Leave names, aliases, hidden state, favorite state, thumbnails, birth dates, and field-source locks unchanged.
