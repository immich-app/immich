# Global Face Identities: Space Matching Metadata Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development for every code task. Use superpowers:verification-before-completion before claiming this phase is done. This is phase 2 of the global face identities roadmap and depends on phase 1.

**Goal:** Use Face Identities in shared-space person matching, let members opt out of metadata contribution before inheritance runs, inherit only permitted metadata into Space People, and merge identities when shared-space evidence proves two source identities are the same real person.

**Architecture:** Shared-space matching waits for source personal identities, then gets or creates `shared_space_person` by `(spaceId, identityId)`. Metadata inheritance copies allowed fields into the space-scoped row only when the source member allows contribution and the target field is empty or inherited. Cross-owner dedupe is handled by identity merge when a shared-space matching or dedupe operation joins faces from different identities into one Space Person.

**Tech Stack:** NestJS, Kysely, `@immich/sql-tools`, BullMQ shared-space jobs, Vitest small and medium tests, OpenAPI SDK generation, Svelte 5 route and component tests.

---

## Scope

This phase owns these umbrella items:

- The shared-space matching and metadata portions of Task 5.
- The metadata, opt-out, and no-leak portions of Task 6.
- The cross-owner shared-space evidence merge path from Task 10.
- The search-performance preparation that affects matching queries and future identity-grouped search, without implementing public global search behavior yet.

This phase intentionally does not add the identity-grouped `/people` page, filter-panel scoped person tokens, global search person rows, or manual repair endpoints. It must leave enough tested identity state for those later phases to dedupe across spaces.
